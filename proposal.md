## Description
My proposal is a sort of character/NPC library generator. This is mostly relevant to creating games but could be useful for things like animations or films.

## Features
### Feature 1: A Searchable Library
This is the primary feature that you can store various character traits such as personality traits, physical appearance, really any aspect of a character.
Each of these individual aspects will get organized under generalized labels Ie. Quick to anger falling under Personality traits.
All of these should be findable via a search function that can use different filters or just simply by name.
### Feature 2: Organization
This is really what enables the library to be cohesive as you can attach labels or tags to different entries so you can do things like browse particular tags.
This obviously enables the searching function but also lets you append tags to entries like adding a "Modern" tag to a firearm entry. From there you can exclude
certain entries from populating when they don't make sense for say a fantasy character or a sci fi character.
### Feature 3: Editable
Pretty self explanetory, you should be able to add your own entries including what tags might be appended to them and what they can fall under to expand the list.
### Feature 4: Visuals
This ideall should have a visual display for it and not simply run out of the terminal. This would be somewhat simialr to the Elden Ring Attribute Randomizor example.
### Feature 5: Character Creator/Generation (Stretch)
This feature will let you dictate particular aspects about a character such as the genre they will be for and then generate a character or NPC that falls within
the conventions of that genre. Obviously some entries are genre agnostic like personality traits but others like clothing, professions, etc can be set for genres.
### Feature 6: Example Characters (Stretch)
There would be a library of popular characters that people know well that has their character attributes and what they are known for broken down via this system.
This would show people what a fleshed out character Ie. Batman would look like within the bounds of the program.
### Feature 7: Exporting/Importing (Stretch)
A way to export things like a small library of your custom tags and entries you've made for say horror genre characters that others could then download online
and import into their program. Also exporting particular characters and their details that you've made.

## Challenges
### Challenge 1: Organization and Optimization
This issue mostly is about how various entries are able to be stored, and referrenced within the program. This gets especially relevant with the various tags
that can be added to a given entry and searching/filtering by tag and such. All of this while trying to keep the program lean and require a minimal amount of
your computer's resources.
### Challenge 2: Visuals
This mostly is a challenge for figuring out how to implement a visual UI for this also ideally one that looks good.
### Challenge 3: Exporting and Importing
Exporting and importing present a particular challenge of how that works, what file type would this need to be and how it gets smoothly added into the program.

## Outcomes
### Ideal Outcome
This is a standalone program that has a bunch of character entries premade and can be exported to anyone, people can then make any additions or edits to the list.
All the features are working and interact smoothly with each other. Ideally this is something I could distribute and build an ecosystem for with sharing and such.
### Minumum Viable Outcome
At minimum the first four features are working as I consider them core features, everything further is extra and is a nice to have. This program should allow you
to quickly search through various entries to be able to be able to build out a character or NPC within given parameters.
